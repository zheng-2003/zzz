docker

一，概念部分：

  • docker是什么？
    • 其实Docker是基于 linux 内核实现，Docker 最早采用 LXC 技术 ，LXC 是 Linux 原生支持的容器技术，可以提供轻量级的虚拟化 ，最初是 dotCloud 公司（后由于 Docker 开源后大受欢迎就将公司改名为 Docker Inc ，总部位于美国加州的旧金山）内部的一个开源的 PAAS 服务 (Platform as a ServiceService )的业余项目。
  • docker的优点：
    • 快速部署：短时间内可以部署成百上千个应用，更快速交付到线上
    • 高效虚拟化：不需要额外hypervisor支持，基于linux内核实现应用虚拟化，相比虚拟机大幅提高性能和效率
    • 节省开支：提高服务器利用率，降低IT支出
    • 简化配置：将运行环境打包保存至容器，使用时直接启动即可
    • 环境统一：将开发，测试，生产的应用运行环境进行标准化和统一，减少环境不一样带来的各种问题
    • 快速迁移和扩展：可实现跨平台运行在物理机、虚拟机、公有云等环境，良好的兼容性可以方便将应用从A宿主机迁移到B宿主机，甚至是A平台迁移到B平台

二，应用场景：

  • 节省项目部署时间
  • 整体打包
  • 配置linux环境等等

    • /bin/bash：放在镜像名后的是命令，这里我们希望有个交互式 Shell，因此用的是 /bin/bas

    dockerfile

  • 每一行以Dockerfile的指令开头，指令不区分大小写，但是惯例使用大写
  • 使用 # 开始作为注释
  • 每一行只支持一条指令，每条指令可以携带多个参数
  • 指令按文件的顺序从上+
  • 至下进行执行
  • 每个指令的执行会生成一个新的镜像层，为了减少分层和镜像大小，尽可能将多条指令合并成一条指令
  • 制作镜像一般可能需要反复多次，每次执行dockfile都按顺序执行，从头开始，已经执行过的指令已经缓存，不需要再执行，后续有一行指令没执行过，再往后的指令将会重新执行，所以为加速镜像制作，将最常变化的内容放下dockerfile的文件的后面

Python基础部分

1.python2 和 python3的区别?

  • python2默认字节是ascii,而python3中是utf-8

    ASCII是每一个数字都有一个数值,UTF-8是不定长编码

  • python2中print是语句,而python3中print是方法

    语句就是不用括号括起来的,方法是用括号括起来耳朵

  • python2中map返回的是一个列表,而python3中返回的是一个可迭代对象

  • python2中默认字节数是4,python3中是2,

  • python2中除法是整形,python3中是一个浮点型

2.您能列举几个PEP8规范吗?

  • RFC(Request For Comments): 专门来定义网络请求的标准文档
  • PEP8:空格缩进

    (每一级缩进4个空格)

  • 导包的时候要分开导,不建议同行导入多个模块
  • 变量名不能以数字,字母,下划线开头
  • 类名定义用首字母大写,函数定义首字母用小写

3.什么是深浅拷贝?

  • 浅拷贝:copy.copy()

    只拷贝第一层,修改深层次两个变量就会互相影响了

  • 深拷贝:copy.deepcopy()

    深拷贝呢,他又叫递归性质的浅拷贝,拷贝完之后修改任意一个变量都不会影响另一个变量

4.Python的自省

  • dir: 获取当前对象内部的属性
  • type: 查看当前对象的类型
  • hasattr: 判断当前对象是否包含某个属性
  • getattr: 获取当前对象的属性
  • isinstance: 判断对象是否是某个类的实例

5.您知道Python中的可变不可变的数据类型吗?

  • 可变的数据类型: 列表(list),字典(dict),集合(set)
  • 不可变数据类型: 数字(number),字符串(string),元祖(tuple),不可变集合(frozenset)

6.您能说说列表和元组的区别吗?

  • 列表的数据可以动态变化而元祖不能

  • 列表一般用于存储同构,列表具有顺序

  • 元祖一般用于存储异构,元祖具有结构

    在数据量大的情况下元祖的操作还是比列表好用的

    7.您能不能说说字典的低层

  • 字典是python中最通用的数据结构之一,他是以键值的方式存放的,他的查询速度比较快,是典型的那空间换时间

  • Cpython中使用了伪随机探测的散列表作为字典的低层数据结构,只有可哈希的对象才能作为字典的键

  • 字典的三个基本操作(添加,获取,删除)的平均时间复杂度为O(1),但是他们的平摊最坏情况复杂度要高的多,O(n)

  • 开放寻址法 : 字典在python3.6之前都是无序的,使用开放寻址法,实现散列表,所以在它产生哈希冲突时,他会用探测函数来去找,直到找到一个空槽来存放待插入的元素

    散列表就是哈希表

    哈希冲突:进行hash运算之后得到了位置进行存储,但是这个位置已经被占用了

  • 拉链法:在python3.6之后,字典就变成了有序的,低层不再是散列表,而是拉链法,它不存在之前开放寻址法中的哈希冲突,遇到冲突他会直接追加到当前位置的尾部,,除非在极端的情况下,可能冲突都进入同一链表,导致当前遍历查询元素的时间较长,解决方案就是,升级为红黑树加快查询速度

    红黑树又称平衡二叉树

    • 根节点是黑色的节点,叶子节点是不存储数据的黑色空节点
    • 任何相邻的节点不能有用一个颜色
    • 任意节点到其可达到的叶子节点间包含相同数量的黑色节点

  • 5.str.strip: 默认移出字符串首位的空白字符串

9.列表内置的方法

  • 2.list.extend:合并两个列表
  • 3.list.count:计算列表中元素的个数
  • 4.list.index:查询列表中某一个元素出现的位置
  • 5.list.insert:将指定元素插入到指定位置上

    10.字典内置的方法

  • 1.dict.keys: 查询所有键
  • 2.dict.values: 查询所有的值
  • 3.dict.item:查询所有的键值对
  • 4.dict.clear:清空字典中的元素
  • 5.dict.copy:拷贝一个新的字典

11.Python中内存管理(垃圾回收机制)

引用计数

引用计数就是当某一个变量被引用的时候,他的引用次数会加1,当引用计数变为0的时候,他将会被释放,当删除一个变量时,,他的引用计数就会减1,但是也会出现,两个变量相互引用,造成无法回收

遇到这种情况可以用,标记-清除解决

标记-清除

标记就是他会自动给活跃的对象添加标识,添加标识的对象不会被清除
清除就是,他会对内存从头到尾的遍历一遍,然后讲那些没有被标识的对象进行内存,将那些原有的对象的标识抹掉,以便下次回收

​ 他最大的缺点就是在垃圾收集后可能有大量的内存碎片,其次就是需要遍历的内存节点比较多,每次回收都将花费大量时间

遇到这种情况可以使用分代回收

分代回收(gc模块):

分代回收就是一种典型的以空间换时间的方法,他会将回收对象分为年轻代,中年代,老年代,每一个代都是一个链表,当python触发垃圾回收机制的话,他会将那些没有被标记的对象进行回收,然后剩下的依次推到中年代,老年代

你可以通过设置gc模块,默认执行700次青年代执行一次中年代,执行10次中年代执行1次老年代,这个可以自己设置,但是切记,不要随意改动,尽量让他间隔大一点,不然会导致他的回收频率过高

Python中的内置方法:

  • map函数:将条件依次作用在列表中的每一个元素上,最后返回一个新的可迭代对象

  • reduce 函数:累计运算的,接收两个参数,这两个参数的结果作为下一趟参数的第一个值

  • filter 函数: 起到一个过滤的作用,将你列表中的元素传给fifter,然后判断结果为True或者Flase将发回True的元素放在一个新的迭代器中

  • sorted 函数:对可迭代对象进行排序,可选参数key(绝对值)为排序方法,也可以通过reverse逆置

微博三方登录

一,微博三方登录特点:

  • 微博登录包括身份认证、用户关系以及内容传播。允许用户使用微博帐号登录访问第三方网站，分享内容，同步信息。它有如下特点：
  • 直接用微博账号登录你的应用或网站；
  • 获得高质量微博活跃用户；
  • 更便捷的分享到微博，提高网站信息传播力；

    二,应用场景:

  • 可以避免复杂繁琐的注册流程,让用户用自己已有的三方账号去登陆

    三,.功能实现

  • 首先进入微博开放平台 https://open.weibo.com/ 注册一个应用(个人,企业),填写应用信息及回调地址
